======================================================================
WINDOWS FILE-LOCK HANDLING
======================================================================

This repository is developed on Windows.

If EPERM, Access Denied, or file-lock errors occur:

1. Identify the exact file.
2. Identify likely stale processes.
3. Do not repeatedly retry the same write indefinitely.
4. Continue unrelated implementation.
5. Stop stale Node, Vitest, Next.js, Python, formatter, or Codex child processes only when safe.
6. Delete only generated caches such as .next when appropriate.
7. Preserve working source changes.
8. Retry validation after the lock is released.
9. Do not restart the entire phase.
10. Report unresolved operating-system issues honestly.

Avoid running development servers during formatting and production builds.
